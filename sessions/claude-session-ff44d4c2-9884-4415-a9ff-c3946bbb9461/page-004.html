Based on my search of the Claude Code documentation and issue tracking, here's a comprehensive answer to your questions about `isolation: "worktree"`:

## How `isolation: "worktree"` Works

When you add `isolation: "worktree"` to an Agent's frontmatter, Claude Code creates a fresh git worktree for that subagent to work in. This allows multiple agents to work in parallel without interfering with each other's file changes. Each subagent gets its own isolated checkout with its own files and branch, while sharing the same repository history and remotes.

The worktree is automatically cleaned up when the agent finishes without making changes. If changes exist, Claude prompts you whether to keep or remove the worktree.

## Configuration in settings.json

For **git repositories**, no explicit settings are needed—worktree isolation works automatically via Claude Code's built-in `git worktree add` logic.

However, if you want custom control over worktree creation (e.g., for non-git VCS), you can configure these hooks:

```json
{
  "hooks": {
    "WorktreeCreate": [
      {
        "type": "command",
        "command": "/path/to/create-worktree.sh"
      }
    ],
    "WorktreeRemove": [
      {
        "type": "command",
        "command": "/path/to/cleanup-worktree.sh"
      }
    ]
  }
}
```

## Known Limitations and Requirements

Based on the official documentation and issue tracking, there are several known limitations:

1. **Git repository detection issue**: On Windows, Claude Code may falsely report "not in a git repository" due to path resolution mismatches, even when the directory is a valid git repo ([Issue #40164](https://github.com/anthropics/claude-code/issues/40164)).

2. **Git check runs before hooks**: When using non-git VCS (SVN, Perforce, Mercurial) with WorktreeCreate hooks configured, the git repository check runs before the hook is considered, preventing the hook from firing ([Issue #32974](https://github.com/anthropics/claude-code/issues/32974)).

3. **In-session EnterWorktree tool ignores hooks**: There's a known bug ([Issue #36205](https://github.com/anthropics/claude-code/issues/36205)) where the in-session Agent tool's `isolation: "worktree"` setting ignores WorktreeCreate/WorktreeRemove hooks and tries to use built-in git worktree logic instead. This is a workaround: use `--worktree` from the CLI (which respects hooks) rather than the Agent tool with `isolation: "worktree"`.

4. **No remote requirement**: Worktrees branch from `origin/HEAD` (your local reference to the remote's default branch), so you need a valid git repository structure locally, but there's no explicit requirement for the remote to exist.

## WorktreeCreate and WorktreeRemove Hooks

These hooks are designed for custom VCS support:

### WorktreeCreate Hook
- **Fires when**: A worktree is being created via `--worktree` flag or `isolation: "worktree"`
- **Input**: JSON on stdin with `worktree_path` and `base_branch`
- **Output**: Must return the absolute path to the created worktree on stdout
- **Exit code**: Non-zero exit aborts creation

Example:
```bash
#!/bin/bash
WORKTREE_PATH=$(jq -r '.worktree_path' < /dev/stdin)
BASE_BRANCH=$(jq -r '.base_branch' < /dev/stdin)

git worktree add "$WORKTREE_PATH" "$BASE_BRANCH"

if [ $? -eq 0 ]; then
  echo "$WORKTREE_PATH"
  exit 0
else
  exit 1
fi
```

### WorktreeRemove Hook
- **Fires when**: A worktree is being removed (at session exit or subagent completion)
